# 团队开发规范

## 1. 目标

这份规范用于统一北极星项目在“产品、设计、研发、验证、发布”上的协作方式。  
当前项目已经进入“后端能力层 + 新前端工作台”并行阶段：

- 后端能力层：`FastAPI + SQLAlchemy + PowerShell`
- 前端正式基线：`frontend/`
- 前端技术栈：`React + Next.js + TailwindCSS + TypeScript + shadcn/ui`

后续所有 Web 界面联合开发，以 `frontend/` 为正式基线。

## 2. 开发入口

- 协作入口：`CLAUDE.md`
- 团队规范：`docs/team-development-standard.md`
- 新前端运行说明：`frontend/README.md`
- 项目整体说明：`README.md`
- 方法模板库：`vendor/vive-md/`

## 3. 前端基线与约束

### 3.1 正式基线

以下任务默认必须在 `frontend/` 中完成：

- 新增 Web 页面
- 现有页面重构
- 视觉升级
- 交互重做
- 页面切换与动画优化
- 团队联合维护的前端模块

旧模板页仅保留为历史能力层参考。  
如遇紧急修复旧页面，可临时修改，但必须补充说明：

- 为什么没有直接改到新前端
- 后续是否要同步迁移

### 3.2 UI 设计规范

所有新前端页面必须遵循：

- 风格：`shadcn/ui` 取向，极简、干净、现代、圆角
- 布局：增加容器内边距和模块间距，保证呼吸感
- 色彩：黑白灰为主色调，仅 CTA 按钮使用浅蓝色
- 排版：强化文字层级，对次要信息统一使用灰色
- 细节：卡片、表格、输入区、弹层必须带轻边框和微阴影
- 过渡：页面切换、导航跳转、登录成功跳转必须带自然缓冲

### 3.3 统一体验要求

禁止出现以下情况：

- 同一产品同时存在两套明显不同的视觉语言
- 继续在旧模板页上叠“新壳层”
- 新页面直接复制旧页面结构，只换颜色
- 页面信息过密，没有留白、分组和层级

## 4. 架构协作规则

### 4.1 前后端职责

- FastAPI 继续承担业务能力层与数据接口
- Next.js 前端负责产品化界面、交互和新体验基线
- 认证、转发、聚合能力优先在 `frontend/src/app/api/*` 中统一处理

### 4.2 新接口流程

需要新增前端能力时，优先按以下顺序推进：

1. 明确页面需要什么数据
2. 定义接口契约或代理结构
3. 确认权限与登录态处理
4. 落页面实现
5. 补验证说明

### 4.3 不允许的做法

- 页面直接散落调用多个后端地址而不收敛到代理层
- 未确认契约就开始并行写多个人的页面
- 一边做新前端，一边继续默认把旧模板作为视觉基线

## 5. 文档先行要求

以下改动不能只写代码，必须至少补一份文档：

- 新业务模块
- 认证或会话逻辑调整
- 代理层改动
- 数据模型或接口结构调整
- DataAgent 对接方式变化
- 团队长期维护的 UI 模式

最低文档内容：

- 改了什么
- 为什么这么改
- 影响范围
- 验证方式

## 6. 安全检查要求

涉及以下范围时必须专项自检：

- 登录、登出、Cookie、Session
- 文件上传 / 下载 / 导入 / 导出
- SQL 查询与写入
- 第三方 API 调用
- 调度脚本执行
- DataAgent / AI 代理调用链

至少检查：

- 输入校验
- 权限边界
- 敏感信息不落日志
- 错误信息可追踪但不过度暴露
- 失败路径可安全回退

## 7. 验证要求

### 7.1 前端提交最低验证

前端改动至少完成：

- `npm run lint`
- `npm run build`
- 关键页面手动预览或截图核对

### 7.2 后端或全链路改动最低验证

涉及接口或能力层变更时，至少补充：

- 接口验证
- 登录态验证
- 页面联调验证

## 8. 提交流程

每次提交必须说明：

1. 改了什么
2. 为什么改
3. 影响哪些页面、接口、脚本或数据
4. 做了什么验证
5. 是否需要同步文档、运行说明或交接说明

如果这 5 件事说不清，就不算完整交付。

## 9. AI 协作要求

团队成员使用 Claude Code、Codex、Copilot 等工具时，默认遵循：

- 先读 `CLAUDE.md`
- 再读 `docs/team-development-standard.md`
- Web 任务优先在 `frontend/` 中实现
- 不允许继续把旧模板页当作新界面主入口
- 交付时必须同时补“变更说明 + 验证结果 + 是否影响规范”

## 10. 当前结论

从现在开始，北极星 Web 重构统一采用：

- `React + Next.js + TailwindCSS + TypeScript + shadcn/ui`

而不是继续围绕旧模板壳做增量美化。  
旧模板能力仍可保留，但新体验、新设计和团队联合开发，都以 `frontend/` 为准。
