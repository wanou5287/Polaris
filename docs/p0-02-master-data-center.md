# P0-02 主数据治理中心

## 1. 背景

北极星当前已经有库存映射、经营看板、预测预警、DataAgent 等能力，但业务对象本身仍然分散在多张表和多个页面里：

- SKU 编码、名称、产品线没有统一主档
- 仓库映射只解决了“叫什么”，没有补齐代码、类型和业务属性
- 库存状态缺少统一的状态组、可售/可供给语义
- 渠道和店铺命名未沉淀成公共台账

这会直接影响后续采购到货、库存状态流转、预警规则、AI 分析和审计追踪的一致性。因此需要先把“对象治理”立起来。

## 2. 本轮目标

本轮目标不是做复杂的主数据工作流，而是先搭出一版可运营、可维护、可被后续模块复用的主数据治理入口。

交付目标：

- 提供主数据治理中心页面
- 统一维护 SKU、仓库、库存状态、渠道店铺四类主数据
- 承接现有仓库映射和库存状态映射，不推翻历史数据
- 自动从库存/销售历史表补种 SKU 与渠道店铺基础数据
- 保存后触发库存清洗刷新，确保治理结果立即影响下游

## 3. 范围

### In Scope

- `SKU 主数据`
- `仓库主数据`
- `库存状态字典`
- `渠道与店铺台账`
- 页面查询、编辑、启停、排序
- 自动补种与默认归类
- 保存后刷新库存清洗结果

### Out of Scope

- 审批流与多人协同锁
- 主数据变更审计详情页
- 角色级字段权限矩阵
- 供应商、采购单、到货单等更复杂主数据

## 4. 数据模型

本轮涉及四类对象：

- `bi_sku_master`
- `bi_inventory_warehouse_map`
- `bi_inventory_status_map`
- `bi_channel_shop_master`

关键增强点：

- `bi_inventory_warehouse_map` 新增 `warehouse_code`、`warehouse_type`、`platform_owner`、`city`、`is_sellable_warehouse`、`is_reverse_warehouse`
- `bi_inventory_status_map` 新增 `status_group`、`can_sell`、`can_forecast_supply`、`need_quality_check`、`next_default_status`
- `bi_sku_master` 作为统一 SKU 主档
- `bi_channel_shop_master` 作为渠道/店铺主档

## 5. 页面设计

页面名称：`主数据治理中心`

页面结构：

- 顶部摘要区：SKU 数、仓库数、状态数、渠道店铺数、库存清洗最新日期、销售最新日期
- 中间四个治理分区：SKU、仓库、库存状态、渠道店铺
- 顶部操作：返回看板、跳转口径中心、刷新、保存、退出

交互原则：

- 所有分区直接表格编辑，降低维护门槛
- 新增行默认给出最小可用结构
- 保存整表时统一校验关键字段与重复值
- 保存成功后给出刷新库存清洗的反馈结果

## 6. API 设计

### `GET /financial/bi-dashboard/api/master-data`

返回四类主数据和页面摘要。

### `PUT /financial/bi-dashboard/api/master-data`

保存四类主数据，支持：

- 更新已有记录
- 新增主数据
- 将未提交旧记录自动置为停用
- 保存后刷新库存清洗数据

## 7. 验收标准

- 页面可打开并展示四类主数据
- 可新增、编辑、保存 SKU/仓库/状态/渠道店铺
- 仓库和状态的增强字段可被持久化
- 首次进入时可从历史库存/销售表自动补种基础数据
- 保存后库存清洗会重新刷新
- 经营看板、预测预警、后续采购/任务模块可复用这些主数据
