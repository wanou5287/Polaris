# P0-01 指标口径中心

## 1. 背景

当前北极星已经具备经营驾驶舱、图表分析、预测预警、DataAgent 报告等能力，但关键指标的业务口径仍分散在页面文案、图表配置和团队默契里。

这会带来几个问题：

- 同一指标在不同页面解释不一致
- AI 报告和人工会议中的表述无法追溯
- 后续做采购、库存、逆向、补货时缺少统一定义

因此需要先建设一个最小可用的指标口径中心，作为治理底座。

## 2. 本轮目标

本轮只做“可维护、可查询、可作为规则源”的第一版，不做复杂审批流。

交付目标：

- 提供指标口径中心页面
- 提供指标字典数据表
- 支持查询、维护、启停、排序
- 支持版本、生效日期、责任部门、来源表说明
- 预置首批核心指标样例

## 3. 范围

### In Scope

- 指标口径中心页面
- 指标字典 CRUD 基础能力
- 指标启停与排序
- 关键字段维护
- 首批默认指标种子数据

### Out of Scope

- 审批流
- 复杂权限矩阵
- 指标自动血缘分析
- 从图表配置反推生成完整口径

## 4. 数据模型

建议对象：`bi_metric_dictionary`

关键字段：

- `metric_key`：指标编码，系统内唯一
- `metric_name`：指标名称
- `business_domain`：所属业务域
- `owner_role`：责任部门/角色
- `definition_text`：业务定义
- `formula_text`：计算口径/规则说明
- `source_table`：来源表
- `source_fields`：来源字段说明
- `dimension_notes`：适用维度说明
- `version_tag`：口径版本
- `effective_date`：生效日期
- `sort_order`：排序
- `is_enabled`：是否启用
- `created_by / updated_by`
- `created_at / updated_at`

## 5. 页面设计

页面名称：`指标口径中心`

页面结构：

- 顶部摘要：指标总数、启用数、业务域数、最近更新时间
- 左侧主表：维护指标字典
- 右侧说明区：维护原则、使用方式、当前轮次范围

主表字段：

- 启用
- 指标编码
- 指标名称
- 业务域
- 责任部门
- 版本
- 生效日期
- 业务定义
- 计算口径
- 来源表
- 来源字段
- 维度说明
- 排序

## 6. API 设计

### `GET /financial/bi-dashboard/api/metric-dictionary`

返回指标字典列表与摘要。

### `PUT /financial/bi-dashboard/api/metric-dictionary`

保存整表内容，支持：

- 更新已有指标
- 新增指标
- 将未提交行自动置为停用

## 7. 验收标准

- 页面可打开并显示默认指标
- 可新增、编辑、保存指标
- 可按关键字和业务域筛选
- 保存后再次进入页面能看到结果
- 现有驾驶舱/预警/AI 报告团队成员能把关键指标追溯到该页面
