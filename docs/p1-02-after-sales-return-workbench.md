# P1-02 逆向售后与退货承接

## 1. 目标

把退货收件、售后诊断、翻新承接、退款闭环与退货拆包观察统一收敛到北极星新工作台，避免逆向链路继续散落在多个旧入口里。

这一轮的落点不是做复杂审批，而是先把逆向执行入口产品化：

- 有统一工单对象
- 有统一状态与责任人
- 能进入任务中心
- 有审计留痕
- 能看到逆向仓与退货拆包的承接快照

## 2. 本轮实现

### 后端

- 新增表：`bi_after_sales_case`
- 新增接口：
  - `GET /financial/bi-dashboard/api/after-sales-workbench`
  - `POST /financial/bi-dashboard/api/after-sales-workbench/cases`
- 默认种子工单覆盖：
  - 退货退款
  - 仅退款
  - 换货
  - 退回翻新
- 查询接口会同时返回：
  - 工单摘要
  - 工单列表
  - 退货拆包观察数据
  - 逆向仓选项
  - 渠道选项

### 平台闭环

- 新模块接入任务中心，来源模块为 `after_sales`
- 新增任务分类：
  - `after_sales_intake`
  - `after_sales_processing`
  - `after_sales_blocker`
- 保存工单后自动触发任务中心同步
- 保存动作写入审计日志

### 前端

- 新工作台路由：`/operations/after-sales-returns`
- 页面包含：
  - 摘要卡片
  - 多条件筛选
  - 工单清单
  - 工单编排表单
  - 逆向执行快照
- 左侧导航与总览快链已接入该入口

## 3. 工单模型

当前首版字段包括：

- 工单号
- 订单号
- 逆向类型
- 售后状态
- 优先级
- 渠道 / 店铺
- SKU
- 申请数量 / 已收件数量
- 退款金额
- 问题分类
- 逆向仓
- 收件日期 / 承诺完成日期
- 负责人 / 角色
- 客户原因
- 诊断结果
- 处理动作
- 备注

## 4. 状态口径

当前使用的售后状态：

- `submitted`：待收件
- `received`：已收件
- `diagnosing`：质检诊断
- `refurbishing`：翻新处理中
- `refund_pending`：待退款
- `closed`：已闭环
- `blocked`：阻塞异常

## 5. 与现有模块关系

- 与主数据治理复用逆向仓配置
- 与任务中心共享统一待办跟踪
- 与审计日志共享操作留痕
- 与翻新协同通过“退回翻新”工单类型形成承接关系
- 与退货拆包出勤复用已有观察数据

## 6. 后续可继续补强

- 售后责任判定与供应商追偿

- 审批流与升级规则
- 与对账补偿中心的自动联动
