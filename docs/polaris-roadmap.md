# 北极星实施路线图

## 1. 目标

北极星不是单纯的 BI 看板，而是一套围绕供应链经营分析、业务动作承接和流程协同持续演进的平台。

当前路线图的目标分三层：

- 先补治理底座，统一口径、对象和审计
- 再补动作型入口，让平台开始承接真实业务执行
- 最后把录入、校验、任务、异常补偿串成闭环

## 2. 当前判断

北极星已经具备这些基础：

- 用友库存和销售数据同步、回填、定时调度
- 经营总览、图表分析、BI 配置和 DataAgent
- 出勤、翻新生产、预测预警、库存映射等业务录入入口
- 钉钉通知和基础运维脚本

当前重点已经从“继续补页面”切换到了“让平台承接真实动作”。因此 P0 的核心是把采购到货、库存流转、任务协同逐步拉进统一工作台。

## 3. 阶段节奏

### 阶段 A：治理底座

- `P0-01` 指标口径中心
- `P0-02` 主数据治理中心
- `P0-03` 审计日志中心

### 阶段 B：动作型入口

- `P0-04` 采购到货协同
- `P0-05` 库存状态流转与调拨触发
- `P0-06` 任务中心与异常待办

### 阶段 C：闭环协同

- `P0-07` 数据对账与失败补偿
- `P1-01` 翻新生产协同升级
- `P1-02` 逆向售后与退货承接
- `P1-03` 补货建议与计划协同

## 4. 当前进度

已完成：

- `P0-01` 指标口径中心
- `P0-02` 主数据治理中心
- `P0-03` 审计日志中心
- `P0-04` 首版采购到货协同
- `P0-05` 首版库存状态流转与调拨触发
- `P0-06` 首版任务中心与异常待办

`P0-04` 当前已落地内容：

- 后端表结构 `bi_procurement_arrival`
- 到货查询与保存 API
- 审计日志自动记账
- 新前端采购到货工作台
- 草稿、待执行、已到货、异常四类状态
- 单据编排状态和异常补偿备注

`P0-05` 当前已落地内容：

- 后端规则表 `bi_inventory_flow_rule`
- 后端任务表 `bi_inventory_flow_task`
- 库存流转查询、规则保存、任务保存 API
- 采购到货保存后自动触发库存流转任务
- 新前端库存流转工作台
- 状态流转 / 仓间调拨两类动作模型

`P0-06` 当前已落地内容：

- 后端统一任务对象表 `bi_task_center_item`
- 采购到货 / 库存流转到任务中心的自动同步
- 任务中心查询与跟进保存 API
- 新前端任务中心工作台
- 总览页任务中心摘要入口
- 任务中心更新审计留痕

## 5. 下一步优先级

建议继续按下面顺序推进：

1. `P0-07` 数据对账与失败补偿
2. `P1-01` 翻新生产协同升级
3. `P1-02` 逆向售后与退货承接

原因：

- 采购到货和库存流转已经形成了第一条动作链
- 任务中心已经把“待执行”和“阻塞”统一收拢
- 下一步最需要的是让阻塞任务具备对账、定位和失败补偿能力
- 没有补偿面板，很多状态仍然停留在人工备注层面

## 6. 仍需你后续补充的信息

这些不是当前实现的阻塞项，但会直接影响后续精度：

- 正式的采购单号编码规则
- 供应商主数据是否需要独立治理
- 到货仓和入库仓是否始终一致
- 单据编排成功 / 失败的真实回写来源
- 库存状态流转是否需要审批节点
- 调拨是否区分申请、审核、执行、回写四段状态
