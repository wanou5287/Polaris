# 北极星实施路线图

## 1. 目标

北极星不是单纯的 BI 看板，而是一套围绕供应链运营分析、动作承接和流程协同持续演进的平台。

当前路线图分三层推进：

- 先补治理底座，统一口径、对象和审计
- 再补动作型入口，让平台开始承接真实业务执行
- 最后把录入、校验、任务、异常补偿串成闭环，并继续延伸到生产、售后和计划协同

## 2. 当前判断

北极星已经具备这些基础：

- 用友库存和销售数据同步、回填、定时调度
- 经营总览、图表分析、BI 配置和 DataAgent
- 出勤、翻新生产、预测预警、库存映射等业务录入入口
- 钉钉通知和基础运维脚本

当前重点已经从“继续补页面”切换到了“让平台承接真实动作”。因此，P0 的核心是把采购到货、库存流转、任务协同和对账补偿逐步拉进统一工作台；P1 的核心则是把这条执行链继续延伸到翻新、售后和计划。

## 3. 阶段节奏

### 阶段 A：治理底座

- `P0-01` 指标口径中心
- `P0-02` 主数据治理中心
- `P0-03` 审计日志中心

### 阶段 B：动作型入口

- `P0-04` 采购到货协同
- `P0-05` 库存状态流转与调拨触发
- `P0-06` 任务中心与异常待办

### 阶段 C：执行闭环

- `P0-07` 数据对账与失败补偿

### 阶段 D：生产与计划协同

- `P1-01` 翻新生产协同升级
- `P1-02` 逆向售后与退货承接
- `P1-03` 补货建议与计划协同

## 5. 下一步优先级

建议接下来按下面顺序推进：

1. `P1-02` 逆向售后与退货承接
2. `P1-03` 补货建议与计划协同

原因：

- 采购、库存、任务、对账、翻新已经形成更完整的执行链
- 现在最值得继续补的是售后逆向入口和补货计划输出
- 后续新模块都可以复用主数据、审计、任务中心和对账补偿底座

## 6. 仍需要补充的信息

这些不是当前实现的阻塞项，但会直接影响后续精度：

- 正式的采购单号编码规则
- 供应商主数据是否需要独立治理
- 到货仓和入库仓是否始终一致
- 单据编排成功 / 失败的真实回写来源
- 库存状态流转是否需要审批节点
- 调拨是否区分申请、审核、执行、回写四段状态
- 翻新产能是否按机型、工位还是班次维护
- 翻新排产是否需要审批或锁定机制
