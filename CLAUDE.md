# 北极星协作入口

本项目当前存在两层实现：

- 后端能力层：`FastAPI + SQLAlchemy + PowerShell`
- 新前端基线：`frontend/`，使用 `React + Next.js + TailwindCSS + TypeScript + shadcn/ui`

从 2026-03-24 起，`frontend/` 是北极星 Web 界面的正式重构基线。后续所有新的 Web 页面、交互优化、设计升级与团队协作，都应优先落在这套新前端中，而不是继续叠加旧模板壳。

## 阅读顺序

1. `CLAUDE.md`
2. `docs/team-development-standard.md`
3. `frontend/README.md`
4. `README.md`
5. `vendor/vive-md/` 中与当前任务相关的模板

## 当前前端基线

- 新界面目录：`frontend/`
- 技术栈：`Next.js 16 + React 19 + TailwindCSS v4 + TypeScript + shadcn/ui`
- 认证方式：通过 `frontend/src/app/api/auth/*` 与 `frontend/src/app/api/backend/*` 代理现有 FastAPI 能力
- 老模板目录仍保留，但仅作为历史能力层参考，不再作为新的视觉和交互基线

## 强制协作规则

### 1. Web 改动优先落在新前端

以下场景禁止继续优先修改旧模板页来做“新界面”：

- 新增 Web 页面
- 现有页面重构
- 导航、布局、交互、动画、视觉升级
- 团队联合开发的前端任务

如果只是修复旧模板页上的紧急线上 bug，可以短期修补；但同时要补一条迁移说明，说明后续是否需要同步到 `frontend/`。

### 2. 统一 UI 设计基线

所有新前端页面必须遵循以下规则：

- 风格：采用 `shadcn/ui` 取向，极简、干净、现代、圆角
- 布局：所有容器增加明确内边距，模块之间保留呼吸感
- 色彩：黑白灰为主色调，仅 CTA 按钮允许使用浅蓝色强调
- 排版：标题、正文、说明信息层级清晰；次要文字统一灰色
- 细节：卡片、表格、表单、弹层必须有微妙边框与轻阴影
- 过渡：页面切换、关键跳转、登录成功跳转都要有自然缓冲，避免突兀闪切

### 3. 先定规则，再写代码

以下改动必须至少补一份文档：

- 新业务模块
- 认证、会话、代理层改动
- 数据模型或接口结构调整
- 团队要长期维护的 UI 模式或设计约束

最低要求：

- 说明改了什么
- 为什么这么改
- 影响哪些页面 / 接口 / 脚本
- 怎么验证

### 4. 团队联合开发时的默认分工

- 后端能力层继续围绕 `app/routes/bi_dashboard.py` 等 FastAPI 模块演进
- 新前端统一在 `frontend/` 目录内开发
- 前端如果需要新接口，优先先补代理层或接口契约，再接页面
- 不允许在同一需求里同时产出两套风格和两套交互语言

## 交付要求

每次提交至少要能回答这 5 件事：

1. 改了什么
2. 为什么改
3. 影响了哪些页面、接口、脚本或数据
4. 做了什么验证
5. 是否需要同步更新文档或运行说明

如果是前端提交，还必须额外说明：

- 是否遵循了新前端基线
- 是否继续使用了 `frontend/`
- 是否满足黑白灰主色、浅蓝 CTA、圆角、间距、轻阴影、自然过渡这些约束
